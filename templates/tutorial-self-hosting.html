‚ö° Quick Start

The fastest way to get started is using Docker Compose. Just clone and run!

# Clone the repository
git clone https://github.com/momokii/go-cli-notes.git
cd go-cli-notes
# Start all services with Docker Compose
docker compose up -d
# Check status
docker compose ps

‚òÄÔ∏è Note: This will start the API server on port 8080 and the web dashboard on port 3000. The database will be initialized automatically.

Step-by-Step Setup üéØ

1

Clone the Repository

git clone https://github.com/momokii/go-cli-notes.git cd go-cli-notes
2

Configure Environment

Copy the example environment file and customize it:

cp .env.example .env

Edit .env with your settings:

# Database Configuration
DB_HOST=postgres
DB_PORT=5432
DB_USER=kg_user
DB_PASSWORD=your_secure_password
DB_NAME=knowledge_garden
# API Server
API_PORT=8080
JWT_SECRET=your_jwt_secret_here
# CORS (if using web dashboard)
WEB_URL=http://localhost:3000
3

Run Database Migrations

Start the PostgreSQL container and run migrations:

# Start only the database first
docker compose up -d postgres
# Run migrations (build migration tool first)
cd internal/migrations go build -o kg-migrate . ./kg-migrate up
# Run migrations (or using straight run from the main.go at migrations folders)
cd internal/migrations go run main.go
4

Start All Services

Start the API server and web dashboard:

docker compose up -d
API Server
http://localhost:8080
Health: /health
Database
postgres:5432
Internal only
5

Configure CLI Client

Configure your CLI to connect to your self-hosted API:

# Register a new account
kg-cli register --email you@example.com --server http://localhost:8080
# Or login if you already have an account
kg-cli login --server http://localhost:8080

üîß Troubleshooting

Common issues and solutions:

Port already in use

Error: bind: address already in use

# Find what's using the port
lsof -i :8080
# Kill the process or change the port in .env
API_PORT=8081
Database connection failed

Error: connection to database failed

# Check if database is running
docker compose ps
# Check database logs
docker compose logs postgres
# Verify credentials in .env
cat .env | grep DB_
Migration errors
# Check migration status
cd cmd/migrate && ./kg-migrate status
# Force rerun migrations
./kg-migrate down ./kg-migrate up
CORS errors in browser

Error: CORS policy: No 'Access-Control-Allow-Origin' header

# Add your frontend URL to .env
WEB_URL=https://your-frontend-domain.com
# Restart the API server
docker compose restart api
Container keeps restarting
# Check container logs
docker compose logs -f api
# Common causes: DB not ready, port conflicts, config errors
